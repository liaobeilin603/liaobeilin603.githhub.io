二、命令

一、命令解释

  1. curl:在命令行或脚本中用于传输数据
  2. ping：测试与另一台计算机之间的连接和延迟
  3. tracert：“tracert主机地址"，可以用来追踪到目的主机地址的路由路径。
  4. wc：在Linux中可以获取指定文件中的字节数，字数，行数，并返回统计结果
  5. grep：用来查看文件包含内容，该命令主要用于查找文件里符合条件的字符串
  6. grep '^tor'：表示文件中查看tor在前面的单词
  7. cut --delimiter=：后面接字符，表示从这个字符分割
  8. cut --delimiter=/ -f1：表示切割文件中所有的“/”，-f1表示提取"/"前面的单词
  9. cut --delimiter=/ -f2：切割（提取）“/”后面的英标
  10. cut --delimiter=/ -f3：切割（提取）“/”后面的意思

  10. grep 'tor$' cet6.txt
  11. cut --help | less
  12. cut --delimiter=/ -f1 cet6.txt
  13. cut --delimiter=/ -f1 cet6.txt | grep 'tor$'
  14. cut --delimiter=/ -f2 cet6.txt
  15. cut --delimiter=/ -f3 cet6.txt

三、总结

gitbash是一个在windows系统上运行Git命令的工具，通过使用命令行实现对计算机的操作，更加接近于计算机的底层。

四、补充:如何从本地上传文件到GitHub上？

  1. step1：初始化本地项目

    2. git init

  2. step2：将文件夹中的文件传至git仓库中

    3. git add

  3. step3：将项目提交到git仓库（commit项目）

    4. git commit -m "注释"

  4. step4：新建分支

    5. git branch -M main

  5. step5：将本地仓库（.git）和gitHub上的仓库进行关联

    6. git remote add origin gitHub仓库地址

  6. step6：将本地仓库的项目push到gitHub的远程仓库上

    7. git push -u origin main
